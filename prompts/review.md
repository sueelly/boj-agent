# 코드 리뷰 지시

사용자가 "리뷰해줘", "제출 완료", "코드 리뷰", "분석해줘", "피드백" 등으로 요청할 때 사용하는 지시문입니다.

---

## 목표

현재 문제 폴더의 **Solution.\*** 만 분석하여, 접근 방식·복잡도·장단점을 정리하고, 대표 풀이와 학습 포인트를 담은 리뷰 내용을 stdout으로 출력합니다. (파일 저장은 CLI가 처리합니다.)

---

## 0. 제출용 코드가 아직 없다면

- 사용자가 "제출해줘"를 하지 않고 "리뷰해줘"만 한 경우, 먼저 **submit/Submit.\*** 를 생성한 뒤(제출용 코드 생성 지시문 참고), 아래 리뷰를 진행할 수 있음.

---

## 1. 분석 대상

- **분석할 파일**: 해당 문제 폴더의 **Solution.\*** 만. (Main, Test, Parse 등은 제외)
- **제출용 코드**: 필요 시 submit/Submit.\* 를 참고해 "백준 제출 형태"와의 일치 여부만 언급 가능.

---

## 2. REVIEW.md 구조

다음 섹션을 포함한 리뷰를 아래 구조대로 출력합니다.

### 2.1 내 코드 분석

```markdown
## 📝 내 코드 분석

### 접근 방식
- 사용한 알고리즘/자료구조
- 핵심 아이디어

### 시간 복잡도
- O(?) — 근거

### 공간 복잡도
- O(?) — 근거

### 장점
- 잘한 점

### 개선 가능한 점
- 더 나아질 수 있는 부분
```

### 2.2 대표 풀이

에이전트가 웹 검색을 지원하면 해당 문제의 대표적/효율적 풀이를 찾아 소개 (선택 사항):

```markdown
## 🏆 대표 풀이

### 접근 방식
- 알고리즘 설명

### 코드
(해당 언어 코드 블록)

### 내 코드와의 차이점
- 어떤 점이 다른지, 왜 이 방식이 효율적인지/다른지
```

### 2.3 다음에 시도해볼 것

```markdown
## 🎯 다음에 시도해볼 것

### 이 문제에서 배울 점
- 핵심 개념/패턴

### 보완할 점
- 더 연습이 필요한 부분

### 관련 문제 추천
- 비슷한 유형 (가능하면)

### 도전 과제
- 더 어려운 제약으로 풀어보기, 다른 방식으로 풀어보기
```

### 2.4 푸터

```markdown
---
리뷰 일자: YYYY-MM-DD
```

---

## 3. 출력 형식

- **제목**: `# [문제번호] 문제제목 - 코드 리뷰` 형태로 시작
- 파일 저장은 CLI가 자동으로 처리하므로, 리뷰 내용만 출력하면 됩니다.

---

**사용자 요청 예**: "리뷰해줘", "제출 완료", "코드 리뷰해줘", "분석해줘", "어땠어?"

이때 Solution만 분석하고, 위 구조대로 리뷰 내용을 출력하면 됨.
