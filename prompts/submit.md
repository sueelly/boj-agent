# 제출용 코드 생성 지시

사용자가 "제출해줘", "백준 제출용 코드 만들어줘" 등으로 요청할 때 사용하는 지시문입니다.

- **언어/확장자**: `~/.config/boj/lang` 또는 `BOJ_LANG` (기본: java). 이에 맞춰 `submit/Submit.*` 확장자와 문법 사용.

---

## 목표

현재 문제 폴더의 **Solution.\*** 에 구현된 알고리즘을 **백준에 그대로 붙여넣어 제출할 수 있는 형태**로 변환한 코드를 생성합니다.

---

## 생성할 파일

- **경로**: `submit/Submit.*` (언어에 맞는 확장자: .java, .py, .cpp 등)
- **내용**: Solution의 로직 + 표준 입출력 처리

---

## 규칙 (백준 환경)

1. **클래스/진입점 이름**: 백준은 클래스명 `Main`(Java), `main` 함수( C++/Python 등)를 요구합니다. 해당 플랫폼 규칙을 따르세요.
2. **패키지/import**: 백준 제출 시 패키지 선언 금지(Java). 필요한 표준 라이브러리만 사용.
3. **입출력**:  
   - Java: `BufferedReader` / `BufferedWriter` 등으로 입출력 처리.  
   - 다른 언어: 해당 언어에서 표준 입력/출력으로 읽고 쓰는 방식으로 통일.
4. **로직**: Solution에 있는 `solve(...)` 로직을 그대로 옮기고, Main(또는 진입점)에서 입력을 읽어 파싱한 뒤 `solve`에 넘겨 호출하고, 결과를 출력하는 형태로 작성.

---

## 작업 순서

1. 현재 문제 폴더의 `Solution.*` 와 `test/Parse.*`(또는 동일 역할 파일)를 확인해 입력 형식을 파악.
2. 동일한 입력 파싱을 제출용 코드 안에서 수행하고, Solution의 알고리즘을 인라인하거나 같은 로직을 Main 안에 구현.
3. `submit/Submit.*` 파일로 저장.

---

**사용자 요청 예**: "제출해줘", "백준 제출용 코드 만들어줘", "Submit.java 만들어줘"

이때 `submit/Submit.*` 한 파일을 생성하면 됨. (REVIEW.md는 생성하지 않음 — 리뷰 요청 시 별도 지시문 사용)
